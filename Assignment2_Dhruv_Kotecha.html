About The Data

I am going to use the csv file from “Amazon Top 50 Bestselling Books 2009 - 2019”. This data set contains 7-Columns. The column names are: Author: The author of the book.

Name of the book.

User rating: Amazon user rating.

Reviews: Number of written reviews on amazon.

Price: The price of the book.

Year: The Year(s) it ranked on the bestseller.

Genre: Whether fiction or non-fiction.

For this case I am choosing R programming, because it allow me work with big amounts of data, cleaning data and create visualizations with little code. Also, I will use R Markdown to reproduce, share my code, and document all the work that I am going to do.

Data set Link- https://www.kaggle.com/sootersaalu/amazon-top-50-bestselling-books-2009-2019

Problem Statement

The data set containing the Amazon Bestselling Books allows us to analyze how, over time, which genres of books did well and which authors sold the most books. We also provide user ratings and reviews for each book, which can help customers narrow down their choices. By doing this visualization, we have a clear representation of the data set as well as an easy comprehension of which author and genre fared best.

Average number of reviews by genre

We can observe that Fiction books have a higher average number of reviews than Non fiction ones. It should be really interesting to know if these reviews are good or bad, because this could help us to analyze if people tend to do more reviews for Fiction books because:

They don’t like it and they want to expose their opinion publicly. People who read Fiction books are more active online and are more likely to do reviews.

User rating vs number of reviews

There isn’t a direct relationship between user rating and number of reviews, but we can see that usually a bigger user rating implies more reviews. It is good to remind that relationship doesn’t imply causation. The min user rating was 3.3. We saw that the book with the lowest user rating was “Casual Vacancy” by J.K Rowling. This book belongs to the Fiction genre.
